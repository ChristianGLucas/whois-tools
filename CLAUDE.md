# christiangeorgelucas/whois-tools ‚Äî Axiom node package

This is an Axiom node package (language: **go**). Axiom turns plain typed
functions into deployed, auto-scaling endpoints ‚Äî you build the whole package
with the `axiom` CLI. This file orients an AI coding agent; follow the loop and
respect the sharp edges. (Generated by `axiom init`; ADR-085.)

## What's here

- `axiom.yaml` ‚Äî the manifest (package name, version, language, nodes, imports, secrets).
- `messages/messages.proto` ‚Äî the protobuf type vocabulary (node inputs/outputs).
- `nodes/` ‚Äî one typed function per node. `gen/` ‚Äî generated bindings.

## The node signature is frozen (ADR-001)

Every node takes `AxiomContext` (the platform handle) plus one typed input and
returns one typed output. For go:

```
func MyNode(ctx context.Context, ax axiom.Context, input *gen.InputMsg) (*gen.OutputMsg, error)
```

## The authoring loop

1. Define messages FIRST ‚Äî `axiom create message InputMsg` / `OutputMsg`, then edit `messages/messages.proto` to add fields.
2. Scaffold each node ‚Äî `axiom create node MyNode --input InputMsg --output OutputMsg`. ALWAYS pass `--input`/`--output` (omitting them prompts interactively and hangs headless).
3. Implement the function body in `nodes/`.
4. `axiom generate` ‚Äî compile messages into `gen/` bindings.
5. Validate ‚Äî `axiom validate --json`. Fix each `ok:false` check (the `detail` tells you how); loop until `passed:true`.
6. Run locally ‚Äî `axiom dev` (serves on :8083, JSON‚Üîprotobuf bridge), then `curl -s localhost:8083/nodes/MyNode -d '{...}'`.
7. Commit AND push to a git remote, then `axiom push --json` (tenant-private) and finally `axiom publish christiangeorgelucas/whois-tools@<version>` (public, immutable).

## Sharp edges (where agents fail)

- **Messages before nodes** ‚Äî `create node` fails if the input/output message doesn't exist yet.
- **Never rely on interactive prompts** ‚Äî always pass `--input`/`--output`; always use `--json` for machine-readable output.
- **PascalCase node names** on the command line; files are snake_case (the CLI handles the file name).
- **Commit AND push before `axiom push`** ‚Äî push builds from the git **remote at HEAD**, not your working tree.
- **Secrets via `ax.secrets.get("KEY")`**, never as node inputs; declare them in the node's `required_secrets`.
- **Nodes can call the internet directly** ‚Äî import an SDK and call out; `ax` is for platform capabilities (secrets, telemetry), not a network jail.
- **`axiom push` is idempotent** (overwrites the tenant-private deploy); **`axiom publish` is immutable** ‚Äî bump the version to re-release.
